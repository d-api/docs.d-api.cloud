## Evento `messages.sent`

<Warning>
  O `messages.sent` é disparado **exclusivamente** para mensagens enviadas via
  **API** da D-API (`/messages/send/*` e `/interactive/send/*`). Mensagens
  enviadas pelo **próprio celular** ou por **outros dispositivos conectados** ao
  mesmo número **não** geram `messages.sent` — elas chegam em
  [`messages.received`](/whatsapp/webhooks/messages-received) com `fromMe: true`.
</Warning>

Use este evento para confirmar que uma mensagem disparada pela API foi enviada
com sucesso e para registrar o conteúdo final entregue ao WhatsApp.

<Note>
  O payload tem **exatamente a mesma estrutura** do evento `messages.received`
  para o mesmo `type` (mesmos campos, `media_data` com `file_length`/`mimetype`,
  `context_info` etc.), apenas com `fromMe: true`. Para a referência completa de
  todos os campos, consulte a página de
  [messages.received](/whatsapp/webhooks/messages-received).
</Note>

A diferença está nos papéis: `from` é o **seu** número (a sessão) e `to` é o
destinatário.

<Note>
  Quando a mídia é enviada por **URL externa**, o payload pode incluir
  `pending_media_url` (a URL original informada na requisição) dentro de
  `media_data`. Além disso, o `id` da mensagem (gerado pelo whatsmeow) pode vir
  em **letras maiúsculas**.
</Note>

## Outros tipos enviáveis

Os tipos abaixo também podem ser enviados pela API. O formato exato do payload
de `messages.sent` para esses tipos **não está documentado aqui** para evitar
exemplos incorretos — quando enviados, o webhook segue a mesma estrutura
do [`messages.received`](/whatsapp/webhooks/messages-received) correspondente,
com `fromMe: true`.

| Tipo | Endpoint |
|------|----------|
| poll | `POST /messages/send/poll` |
| album | `POST /messages/send/album` |
| nativeflow | `POST /interactive/send/nativeflow` |

<Note>
  Enquetes (`poll`) são reportadas em `messages.received` como `poll_update`
  (votos). O **álbum** envia múltiplas mídias e pode gerar mais de um evento de
  mídia individual (`image`/`video`). Confirme o formato em ambiente de teste
  antes de tratar esses tipos em produção.
</Note>

---

## Endpoints de envio

Cada tipo enviável corresponde a um endpoint da API:

| Tipo | Endpoint |
|------|----------|
| text | `POST /messages/send/text` |
| image | `POST /messages/send/image` |
| video | `POST /messages/send/video` |
| audio | `POST /messages/send/audio` |
| document | `POST /messages/send/document` |
| sticker | `POST /messages/send/sticker` |
| location | `POST /messages/send/location` |
| contact | `POST /messages/send/contact` |
| poll | `POST /messages/send/poll` |
| reaction | `POST /messages/send/reaction` |
| album | `POST /messages/send/album` |
| list | `POST /interactive/send/list` |
| carousel | `POST /interactive/send/carousel` |
| nativeflow | `POST /interactive/send/nativeflow` |

<Warning>
  Evite usar `messages.sent` para acionar automações de resposta automática.
  Como ele confirma mensagens que **você mesmo** enviou pela API, tratá-lo como
  gatilho de resposta pode gerar loops.
</Warning>
